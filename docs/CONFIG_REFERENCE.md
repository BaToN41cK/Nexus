## Файлы конфигурации

| Файл | Назначение | Формат |
|------|------------|--------|
| `config/nexus.yaml` | Шаблон по умолчанию (в репозитории) | YAML |
| `~/.nexus/config.yaml` | Пользовательская конфигурация | YAML |
| `~/.nexus/.env` | API-ключи и секреты | KEY=VALUE |
| `config/.env` | Локальный .env (для разработчиков) | KEY=VALUE |

> При первом запуске Nexus создаёт `~/.nexus/config.yaml` из шаблона `config/nexus.yaml`.

## Поиск конфигурации

### config.yaml

| Приоритет | Путь | Описание |
|-----------|------|----------|
| 1 | Путь из флага `--config` | Явное указание |
| 2 | `./config/nexus.yaml` | Текущая рабочая директория |
| 3 | `~/.nexus/config.yaml` | Домашняя директория пользователя |
| 4 | `config/nexus.yaml` | Внутри пакета (дефолтный шаблон) |

### .env файла

| Приоритет | Путь | Описание |
|-----------|------|----------|
| 1 | Путь из `NEXUS_ENV_PATH` | Явное указание |
| 2 | `~/.nexus/.env` | Домашняя директория |
| 3 | `./config/.env` | Текущая рабочая директория |
| 4 | Поиск `config/.env` вверх по дереву каталогов | Рекурсивный поиск |
| 5 | Системные переменные окружения | `GROQ_API_KEY`, `OPENAI_API_KEY`, `ANTHROPIC_API_KEY` |

## Основные параметры

```yaml
# --- Провайдер ---
provider: "groq"                          # groq | openai | anthropic | ollama
groq_model: "llama-3.3-70b-versatile"    # Модель для выбранного провайдера
base_url: ""                              # Для OpenAI-совместимых API и Ollama
```

| Параметр | Тип | Дефолт | Описание |
|----------|-----|--------|----------|
| `provider` | `string` | `"groq"` | LLM-провайдер: `groq`, `openai`, `anthropic`, `ollama` |
| `groq_model` | `string` | `"llama-3.3-70b-versatile"` | Модель для провайдера (используется как «основная модель» для всех провайдеров) |
| `base_url` | `string` | `""` | Базовый URL для OpenAI-совместимых API и Ollama (по умолчанию `http://localhost:11434` для Ollama) |

### Доступные модели

| Провайдер | Модель по умолчанию | Примеры других моделей |
|-----------|---------------------|------------------------|
| groq | `llama-3.3-70b-versatile` | `mixtral-8x7b-32768`, `llama-3.1-8b-instant`, `gemma2-9b-it` |
| openai | `gpt-4o` | `gpt-4o-mini`, `gpt-3.5-turbo`, `gpt-4-turbo` |
| anthropic | `claude-sonnet-4-20250514` | `claude-haiku-3-5`, `claude-opus-4-20250514` |
| ollama | `llama3.2` | `mistral`, `codellama`, `phi3`, `qwen2.5` |

## Параметры генерации

```yaml
# --- Генерация ---
timeout: 30                               # Таймаут запроса (секунды)
max_tokens: 4096                          # Максимальное количество токенов в ответе
temperature: 0.7                          # Температура сэмплирования (0.0–2.0)
```

| Параметр | Тип | Дефолт | Диапазон | Описание |
|----------|-----|--------|----------|----------|
| `timeout` | `int` | `30` | 5–300 | Таймаут запроса к API (секунды) |
| `max_tokens` | `int` | `4096` | 1–128000 | Максимальное количество токенов в ответе |
| `temperature` | `float` | `0.7` | 0.0–2.0 | Температура сэмплирования. 0.0 = детерминированный ответ, 2.0 = максимальная креативность |

### Рекомендации по temperature

| Задача | Рекомендуемое значение |
|--------|------------------------|
| Код, факты, математика | 0.0–0.3 |
| Общий диалог | 0.5–0.7 |
| Креативное письмо, мозговой штурм | 0.8–1.2 |
| Эксперименты | 1.5–2.0 |

## Параметры загрузки контента

```yaml
# --- Загрузка контента ---
max_content_length: 50000                 # Макс. длина подгружаемого контента (символов)
summarize_threshold: 40000                # Порог автоматической суммаризации
```

| Параметр | Тип | Дефолт | Описание |
|----------|-----|--------|----------|
| `max_content_length` | `int` | `50000` | Максимальная длина контента из URL (символов). Контент обрезается до этого значения |
| `summarize_threshold` | `int` | `40000` | Если контент длиннее этого порога, Nexus автоматически делает суммаризацию перед отправкой в LLM |

## Параметры кэша

```yaml
# --- Кэш ---
cache_ttl: 3600                           # TTL кэша в секундах
max_cache_size_mb: 50                     # Авто-очистка кэша по размеру (MB)
max_retries: 3                            # Количество повторных попыток
rate_limit: 5                             # Лимит запросов
```

| Параметр | Тип | Дефолт | Описание |
|----------|-----|--------|----------|
| `cache_ttl` | `int` | `3600` | Время жизни кэша (секунды). После истечения запись считается устаревшей |
| `max_cache_size_mb` | `int` | `50` | Максимальный размер кэша в МБ. При превышении автоматически удаляются старые записи |
| `max_retries` | `int` | `3` | Количество повторных попыток при ошибках API |
| `rate_limit` | `int` | `5` | Лимит запросов к API (в секунду или за период) |

## Параметры диалога

```yaml
# --- Диалог ---
conversation_history_size: 5              # Сколько прошлых обменов помнить
system_prompt: "Ты — полезный ассистент. Отвечай кратко и по делу."
```

| Параметр | Тип | Дефолт | Описание |
|----------|-----|--------|----------|
| `conversation_history_size` | `int` | `5` | Количество последних обменов репликами, включаемых в контекст LLM |
| `system_prompt` | `string` | `"Ты — полезный ассистент. Отвечай кратко и по делу."` | Системный промпт для LLM (определяет поведение ассистента) |

### Рекомендации по system_prompt

| Задача | Пример system_prompt |
|--------|---------------------|
| Общий ассистент | `"Ты — полезный ассистент. Отвечай кратко и по делу."` |
| Программист | `"Ты — опытный программист. Отвечай на коде, приводи примеры."` |
| Переводчик | `"Ты — профессиональный переводчик. Переводи точно, сохраняя стиль."` |
| Учитель | `"Ты — терпеливый учитель. Объясняй простыми словами, приводи примеры."` |
| Рецензент | `"Ты — редактор. Проверяй текст на ошибки и предлагай улучшения."` |

## Параметры web-поиска

```yaml
web_search:
  enabled: false               # Если true — --search подразумевается у nexus run
  backend: "auto"              # auto | duckduckgo | tavily | searxng | bing
  max_results: 5               # Сколько результатов запрашивать у поисковика
  fetch_top_n: 3               # Сколько top-страниц загрузить как контекст
  timeout: 15                  # Таймаут запросов (секунды)
  cache_enabled: true          # Кэшировать результаты поиска
  cache_ttl: 3600              # TTL кэша поиска (секунды)
  # API-ключи (можно указать здесь или через .env):
  # tavily_api_key: "tvly-..."
  # bing_api_key: "..."
  # searxng_url: "https://searx.be"
```

### Общие параметры поиска

| Параметр | Тип | Дефолт | Описание |
|----------|-----|--------|----------|
| `enabled` | `bool` | `false` | Включён ли web-поиск по умолчанию |
| `backend` | `string` | `"auto"` | Поисковый бэкенд: `auto`, `duckduckgo`, `tavily`, `searxng`, `bing` |
| `max_results` | `int` | `5` | Количество результатов поиска (1–20) |
| `fetch_top_n` | `int` | `3` | Количество top-страниц для загрузки как контекст |
| `timeout` | `int` | `15` | Таймаут запросов к поисковику (секунды) |
| `cache_enabled` | `bool` | `true` | Кэшировать результаты поиска |
| `cache_ttl` | `int` | `3600` | TTL кэша поиска (секунды) |

### API-ключи поисковиков

| Параметр | Переменная окружения | Описание |
|----------|---------------------|----------|
| `tavily_api_key` | `TAVILY_API_KEY` | [tavily.com](https://tavily.com) — рекомендуется для LLM |
| `bing_api_key` | `BING_API_KEY` | Azure Bing Web Search |
| `searxng_url` | `SEARXNG_URL` | URL self-hosted или публичного экземпляра SearXNG |
| _не требуется_ | _не требуется_ | DuckDuckGo — работает без API-ключа |

### Приоритет выбора бэкенда (при `backend: "auto"`)

```
tavily → bing → searxng → duckduckgo
```

DuckDuckGo используется как fallback и работает без API-ключа.

## Полный пример конфига

```yaml
# ============================================
# Nexus — Полный конфигурационный файл
# ============================================

# --- Провайдер ---
provider: "groq"                          # groq | openai | anthropic | ollama
groq_model: "llama-3.3-70b-versatile"    # Модель для выбранного провайдера
base_url: ""                              # Для OpenAI-совместимых API и Ollama

# --- Генерация ---

max_tokens: 4096                          # Максимальное количество токенов в ответе

# --- Загрузка контента ---
max_content_length: 50000                 # Макс. длина подгружаемого контента (символов)
summarize_threshold: 40000                # Порог автоматической суммаризации

# --- Кэш ---
cache_ttl: 3600                           # TTL кэша в секундах
max_cache_size_mb: 50                     # Авто-очистка кэша по размеру (MB)
max_retries: 3                            # Количество повторных попыток

# --- Диалог ---

# --- Web-поиск ---

  max_results: 5
  fetch_top_n: 3

  cache_enabled: true

  # API-ключи (можно указать здесь или через .env):

  # bing_api_key: "..."
  # searxng_url: "https://searx.be"
```

## Приоритет параметров

Параметры конфигурации применяются в следующем порядке (от низшего к высшему):

1. **Дефолтные значения** (в коде)
2. **Файл конфигурации** (`config/nexus.yaml` → `~/.nexus/config.yaml`)
3. **Переменные окружения** (`GROQ_API_KEY`, `NEXUS_LANG`, и т.д.)
4. **CLI-флаги** (`--search`, `--no-cache`, `--lang`, `--config`)

**Пример:** Если в конфиге `temperature: 0.7`, но передан флаг `--temperature 0.3`, будет использоваться `0.3`.

## Валидация

Nexus автоматически валидирует параметры конфигурации:

| Параметр | Валидация |
|----------|-----------|
| `provider` | Должен быть одним из: `groq`, `openai`, `anthropic`, `ollama` |
| `timeout` | Целое число, 5–300 |
| `max_tokens` | Целое число, 1–128000 |
| `temperature` | Число с плавающей точкой, 0.0–2.0 |
| `max_content_length` | Целое число, > 0 |
| `summarize_threshold` | Целое число, > 0 |
| `cache_ttl` | Целое число, > 0 |
| `max_cache_size_mb` | Целое число, > 0 |
| `conversation_history_size` | Целое число, > 0 |

При нарушении валидации Nexus выдаёт сообщение `Invalid configuration: ...` и завершается.

## Troubleshooting

### `Invalid configuration: ...`

**Решение:**
1. Проверьте YAML-синтаксис в `~/.nexus/config.yaml`
2. Убедитесь, что типы значений корректны (строки в кавычках, числа без кавычек)
3. Удалите `~/.nexus/config.yaml` — он будет пересоздан с дефолтами

### Конфиг не применяется

**Решение:**
1. Проверьте путь к конфигу: `nexus status`
2. Убедитесь, что файл существует: `ls -la ~/.nexus/config.yaml`
3. Проверьте YAML-синтаксис: `python -c "import yaml; yaml.safe_load(open('$HOME/.nexus/config.yaml'))"`

### Как узнать текущие настройки

```bash
# Показать статус системы (включая пути)
nexus status

# Показать текущий конфиг
cat ~/.nexus/config.yaml
```

### Сброс конфигурации

```bash
# Удалить пользовательский конфиг (будет пересоздан при следующем запуске)
rm ~/.nexus/config.yaml

# Или восстановить из шаблона
cp config/nexus.yaml ~/.nexus/config.yaml
```
