## Обзор

Производительность Nexus зависит от:
1. **Выбранной модели** (самый важный фактор)
2. **Сетевого соединения** (таймауты, стабильность)
3. **Размера контекста** (история + загруженный контент)
4. **Кэширования** (повторные запросы)
5. **Конфигурации** (токены, температура)

## Настройки генерации

### max_tokens

Ограничивает длину ответа. Меньшие значения = быстрее.

```yaml
# Быстрые ответы
max_tokens: 1024

# Стандартные ответы
max_tokens: 4096

# Длинные ответы (статьи, код)
max_tokens: 8192
```

| Задача | Рекомендуемое значение |
|--------|------------------------|
| Короткие ответы | 512–1024 |
| Общий диалог | 2048–4096 |
| Длинные тексты | 4096–8192 |
| Код | 4096–16384 |

### temperature

Влияет на креативность, но не на скорость. Однако низкая температура может давать более предсказуемые ответы.

```yaml
# Детерминированные ответы (код, факты)
temperature: 0.0–0.3

# Общий диалог
temperature: 0.5–0.7

# Креативное письмо
temperature: 0.8–1.2
```

### timeout

Увеличьте при медленном интернете или больших моделях:

```yaml
timeout: 60  # по умолчанию 30
```

## Кэширование

### Включение кэша

```yaml
cache_ttl: 3600  # 1 час
max_cache_size_mb: 50  # 50 МБ
```

### Когда отключать кэш

- Уникальные запросы (кэш не поможет)
- Актуальная информация (нужен свежий ответ)
- Тестирование (проверка нового промпта)

```bash
nexus run "Уникальный вопрос" --no-cache
```

### Оптимизация кэша

1. **Уменьшите TTL** для актуальных данных:
   ```yaml
   cache_ttl: 600  # 10 минут
   ```

2. **Увеличьте размер кэша** для большого объёма:
   ```yaml
   max_cache_size_mb: 200
   ```

3. **Очищайте кэш регулярно**:
   ```bash
   nexus cache-clear
   ```

## История диалога

### conversation_history_size

Чем меньше история = тем быстрее ответ (меньше токенов в контексте).

```yaml
# Быстрые ответы (мало контекста)
conversation_history_size: 3

# Стандартный контекст
conversation_history_size: 5

# Длинный контекст (много истории)
conversation_history_size: 10
```

### Рекомендации

| Сценарий | Рекомендуемое значение |
|----------|------------------------|
| Быстрые одноразовые запросы | 1–3 |
| Общий диалог | 3–5 |
| Длинные сессии | 5–10 |
| Рабочие задачи (код, документы) | 7–15 |

## Загрузка контента

### max_content_length

Ограничивает размер загружаемого контента.

```yaml
# Быстрая загрузка (маленькие страницы)
max_content_length: 10000

# Стандартная загрузка
max_content_length: 50000

# Полная загрузка (длинные статьи)
max_content_length: 100000
```

### summarize_threshold

Автоматическая суммаризация длинного контента.

```yaml
# Суммаризировать длинные тексты
summarize_threshold: 20000

# Загружать полностью (без суммаризации)
summarize_threshold: 100000
```

## Web-поиск

### Оптимизация поиска

```yaml
web_search:
  enabled: true
  max_results: 3        # меньше результатов = быстрее
  fetch_top_n: 2        # меньше загрузок = быстрее
  timeout: 10           # короткий таймаут
  cache_enabled: true   # кэшировать результаты
```

### Когда отключать поиск

- Локальные задачи (без интернета)
- Уникальные запросы (кэш не поможет)
- Тестирование (проверка нового промпта)

```bash
nexus run "Простой вопрос" --no-search
```

## SQLite оптимизация

### PRAGMA настройки

Для больших баз данных можно оптимизировать SQLite:

```python
import sqlite3

conn = sqlite3.connect("~/.nexus/memory.db")
cursor = conn.cursor()

# Увеличить кэш страниц
cursor.execute("PRAGMA cache_size = -64000")  # 64 MB

# WAL режим (параллельное чтение/запись)
cursor.execute("PRAGMA journal_mode = WAL")

# Синхронизация (меньше = быстрее, но менее безопасно)
cursor.execute("PRAGMA synchronous = NORMAL")

# Вакуум (освободить место)
cursor.execute("VACUUM")
```

### FTS5 оптимизация

```python
# Пересоздать FTS индекс
cursor.execute("INSERT INTO exchanges_fts(exchanges_fts) VALUES('rebuild')")
```

## Рекомендации по оптимизации

### Для быстрых ответов

1. Используйте `llama-3.1-8b-instant` (Groq)
2. Уменьшите `max_tokens: 1024`
3. Уменьшите `conversation_history_size: 2`
4. Отключите кэш для уникальных запросов

### Для高质量 ответов

1. Используйте `gpt-4o` (OpenAI) или `claude-sonnet-4-20250514` (Anthropic)
2. Увеличьте `max_tokens: 8192`
3. Увеличьте `conversation_history_size: 10`
4. Включите кэш для повторных запросов

### Для большого объёма

1. Включите кэш: `cache_ttl: 3600`
2. Ограничьте контент: `max_content_length: 50000`
3. Используйте SQLite для истории: `SqliteMemoryStore`
4. Очищайте кэш регулярно: `nexus cache-clear`
