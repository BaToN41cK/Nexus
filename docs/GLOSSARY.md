## A

### Agent (Агент)
Программа, которая может выполнять действия самостоятельно или с помощью инструментов. В Nexus используется `NexusAgent` для общения с LLM и `ReActAgent` для многошагового рассуждения.

### API (Application Programming Interface)
Интерфейс программирования приложений — способ взаимодействия между программами. Nexus использует API провайдеров (Groq, OpenAI, Anthropic) для генерации ответов.

### API Key (API-ключ)
Секретный ключ для доступа к API. Nexus использует `GROQ_API_KEY`, `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`.

---

## B

### Backend (Бэкенд)
Серверная часть приложения. В Nexus используется для обозначения:
- Поисковых бэкендов (DuckDuckGo, Tavily, SearXNG, Bing)
- Бэкендов памяти (JSON, SQLite)

### BaseProvider
Абстрактный базовый класс для всех LLM-провайдеров в Nexus. Определяет интерфейс `generate()` и `generate_stream()`.

---

## C

### Cache (Кэш)
Механизм сохранения результатов для повторного использования. Nexus кэширует ответы LLM и результаты web-поиска.

### CLI (Command Line Interface)
Интерфейс командной строки. Nexus предоставляет CLI для взаимодействия с пользователем.

### Context (Контекст)
Информация, передаваемая LLM для генерации ответа. Включает системный промпт, историю диалога и загруженный контент.

### Content Loader (Загрузчик контента)
Модуль Nexus для загрузки и извлечения текста из различных источников (URL, файлы).

---

## D

### Docker
Платформа для контейнеризации приложений. Nexus предоставляет Dockerfile для запуска в контейнере.

---

## E

### Embedding (Встраивание)
Представление текста в виде вектора (списка чисел). Используется для семантического поиска. Планируется добавить в Nexus.

### Exchange (Обмен репликами)
Один обмен сообщениями между пользователем и ассистентом. Состоит из `prompt` и `response`.

---

## F

### FTS5 (Full-Text Search 5)
Механизм полнотекстового поиска в SQLite. Используется в `SqliteMemoryStore` для быстрого поиска по истории.

### Function Calling
Механизм вызова функций LLM. Позволяет модели вызывать внешние инструменты. Планируется добавить в Nexus.

---

## G

### Groq
LLM-провайдер, предоставляющий быстрый доступ к моделям Llama, Mixtral и другим. Рекомендуется для быстрого старта.

---

## H

### History (История)
Запись всех запросов и ответов Nexus. Хранится в `~/.nexus/history/history.log`.

---

## I

### i18n (Internationalization)
Интернационализация — поддержка множества языков. Nexus поддерживает русский и английский.

### Interactive Mode (Интерактивный режим)
Режим диалога с LLM в реальном времени. Запускается командой `nexus interactive`.

---

## J

### JSON (JavaScript Object Notation)
Формат данных. Используется для хранения истории диалога (`conversation.json`) и конфигурации.

---

## K

### Key (Ключ)
Секретная строка для доступа к сервисам. См. API Key.

---

## L

### LLM (Large Language Model)
Большая языковая модель — модель ИИ, обученная на большом корпусе текста. Nexus работает с моделями Llama, GPT, Claude.

### Local (Локальный)
Режим работы без доступа к интернету. В Nexus доступен через Ollama.

---

## M

### MCP (Model Context Protocol)
Стандарт для подключения ИИ-инструментов к LLM-клиентам. Nexus предоставляет MCP-сервер для Claude Desktop, Cursor, Continue.

### Memory (Память)
Хранилище истории диалогов. В Nexus доступны два бэкенда: JSON и SQLite.

### Middleware
Промежуточное ПО для обработки запросов. Планируется добавить в Nexus.

---

## N

### Nexus
Название проекта — CLI-инструмент для работы с LLM.

---

## O

### Ollama
Платформа для запуска LLM-моделей локально. Позволяет использовать Nexus без API-ключей.

---

## P

### Plugin (Плагин)
Модуль расширения функционала. Планируется добавить систему плагинов в Nexus.

### Prompt (Промпт)
Текстовый запрос к LLM. Может включать системный промпт, историю и контекст.

### Provider (Провайдер)
Сервис, предоставляющий доступ к LLM. В Nexus: Groq, OpenAI, Anthropic, Ollama.

---

## R

### RAG (Retrieval-Augmented Generation)
Генерация с.retrieve из базы знаний. Позволяет LLM использовать внешние документы. Планируется добавить в Nexus.

### ReAct (Reasoning + Acting)
Паттерн многошагового рассуждения с вызовом инструментов. LLM чередует мысли, действия и наблюдения.

### ReactResult
Объект, возвращаемый `ReActAgent.run()`. Содержит `final_answer`, `steps`, `iterations`, `duration_s`.

### Rich
Библиотека для красивого вывода в терминале. Используется в Nexus для Markdown, подсветки кода, прогресс-баров.

---

## S

### SDK (Software Development Kit)
Набор инструментов для разработки. Nexus использует SDK провайдеров (`groq`, `openai`, `anthropic`, `ollama`).

### Search (Поиск)
Механизм поиска информации в интернете. Nexus поддерживает DuckDuckGo, Tavily, SearXNG, Bing.

### SQLite
Легковесная реляционная база данных. Используется в `SqliteMemoryStore` для хранения истории.

### Streaming (Стриминг)
Потоковая передача данных. Nexus поддерживает стриминг ответов LLM.

### System Prompt (Системный промпт)
Инструкция для LLM, определяющая её поведение. Настраивается в конфиге.

---

## T

### Temperature (Температура)
Параметр сэмплирования LLM. 0.0 = детерминированный ответ, 2.0 = максимальная креативность.

### Tool (Инструмент)
Функция, которую LLM может вызвать во время ReAct-цикла. В Nexus: `web_search`, `web_fetch`.

### ToolRegistry
Реестр инструментов для ReAct-агента. Позволяет регистрировать кастомные инструменты.

### TTL (Time To Live)
Время жизни записи в кэше. После истечения запись считается устаревшей.

---

## V

### venv (Virtual Environment)
Виртуальное окружение Python. Изолирует зависимости проекта.

---

## W

### Web Search (Web-поиск)
Поиск информации в интернете. Nexus поддерживает несколько поисковых бэкендов.

### WebSearcher
Фасад для web-поиска в Nexus. Автоматически выбирает бэкенд и кэширует результаты.
