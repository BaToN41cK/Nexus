## Что считается уязвимостью

- Несанкционированный доступ к API-ключам
- Инъекция кода через промпты (prompt injection)
- Утечка данных из кэша или истории
- Командная инъекция через URL
- Несанкционированный доступ к файловой системе
- Denial of Service через обработку контента

## Меры безопасности

### API-ключи

- API-ключи хранятся в `.env` файле (не в коде)
- `.env` добавлен в `.gitignore` (не коммитится)
- Ключи передаются через переменные окружения
- Рекомендуется использовать ключи с ограниченными правами

### Ввод пользователя

- URL проверяются через `urlparse` перед загрузкой
- Файлы загружаются с ограничением размера (`max_content_length`)
- Контент обрезается перед отправкой в LLM
- Ошибки обрабатываются без раскрытия внутренностей системы

### Кэш

- Кэш хранится локально в `~/.nexus/cache/`
- Кэш автоматически очищается при превышении размера
- TTL предотвращает использование устаревших данных
- Результаты web-поиска кэшируются отдельно

### MCP-сервер

- Работает через stdio (не принимает сетевые подключения)
- API-ключи передаются через переменные окружения
- Сервер не логирует запросы и ответы

### Зависимости

- Используйте только официальные пакеты
- Регулярно обновляйте зависимости
- Проверяйте зависимости на известные уязвимости:

```bash
pip install safety
safety check
```

## Рекомендации для пользователей

1. **Не коммитьте API-ключи** в репозиторий
2. **Используйте переменные окружения** для хранения секретов
3. **Ограничивайте права API-ключей** (только необходимые операции)
4. **Регулярно обновляйте Nexus** и зависимости
5. **Проверяйте URL** перед загрузкой контента
6. **Используйте виртуальные окружения** для изоляции зависимостей
